주요 기술 스택
==============

백엔드:
- TypeScript
- Cloudflare Workers
- Durable Objects (상태 관리)
- KV 저장소 (PDF 파일 저장)

프론트엔드:
- HTML5
- CSS3 (반응형)
- Vanilla JavaScript (WebSocket API)
- WebRTC API
- PDF.js (PDF 렌더링)

통신:
- Native WebSocket (Socket.IO 제거)
- JSON 메시지 프로토콜
- 자동 재연결 로직

주요 엔드포인트
===============

WebSocket:
- ws://host/ws/signaling?roomId=xxx
- ws://host/ws/video-call?roomId=xxx

REST API:
- GET /api/health
- GET /api/turn-config
- POST /api/video-call/upload-pdf
- GET /api/video-call/pdf-list

정적 자산:
- /index.html
- /signaling/index.html
- /video-call/index.html
- /video-call/css/style.css

- /turn-relay/index.html


성능 특성
=========

메모리: Durable Objects의 효율적 관리
처리량: HTTP/3 기반 CDN 에지 처리
레이턴시: 전 세계 ~100ms 이하
확장성: 방별 자동 분산
제한: 시그널링 2명, 비디오콜 10명/방


Socket.IO → WebSocket 마이그레이션 포인트
==========================================

변경 사항:
1. socket.emit() → sendWsMessage({type:"...", data:{...}})
2. socket.on() → ws.onmessage 핸들러
3. 자동 재연결 로직 수동 구현
4. 네임스페이스 → 경로 파라미터 (roomId)
5. 파일시스템 → KV 저장소 (PDF)

이점:
- 더 가벼운 클라이언트 (소스 맵 제거)
- 더 빠른 연결 (3-way handshake만 필요)
- 더 간단한 디버깅 (표준 WebSocket API)
- 비용 절감 (Socket.IO 라이브러리 불필요)
